## 2. Sitemap 제출

### 2-1. Sitemap 메뉴 접속

**경로**: 좌측 메뉴 → "Sitemaps" (색인 섹션 하단)

**화면 위치**:
```
좌측 메뉴
├─ 개요
├─ 실시간 보고서
├─ 성능
├─ URL 검사
├─ 색인 생성
│  ├─ 페이지
│  ├─ Sitemaps ← 여기를 클릭
│  └─ 제거됨
└─ 보안 및 수동 조치
```

### 2-2. 새 Sitemap 제출

**단계 1**: "새 사이트맵 제출" 버튼 클릭
- 화면 우상단의 **"새 사이트맵 제출"** 버튼 클릭

**단계 2**: Sitemap URL 입력
```
입력 필드에 다음 URL 입력:
https://muunsaju.com/sitemap.xml
```

**단계 3**: "제출" 버튼 클릭
- 파란색 "제출" 버튼 클릭

**화면 예시**:
```
┌──────────────────────────────────────────────┐
│ Sitemaps                                     │
├──────────────────────────────────────────────┤
│                                              │
│ 새 사이트맵 제출 [버튼]                      │
│                                              │
│ 사이트맵 URL 입력:                           │
│ ┌────────────────────────────────────────┐  │
│ │ https://muunsaju.com/sitemap.xml       │  │
│ └────────────────────────────────────────┘  │
│                                              │
│                          [제출] [취소]       │
└──────────────────────────────────────────────┘
```

### 2-3. 제출 확인

**성공 메시지**:
```
✅ Sitemap이 성공적으로 제출되었습니다.
```

**제출된 Sitemap 목록**:
- 상태: "성공" (녹색 체크)
- URL: `https://muunsaju.com/sitemap.xml`
- 발견된 URL: 49개 (또는 최신 개수)
- 색인된 URL: 0개 → 시간 경과에 따라 증가

## 3. URL 색인 요청

### 3-1. URL 검사 도구 접속

**경로**: 좌측 메뉴 → "URL 검사"

**화면 위치**:
```
좌측 메뉴
├─ 개요
├─ 실시간 보고서
├─ 성능
├─ URL 검사 ← 여기를 클릭
├─ 색인 생성
└─ 보안 및 수동 조치
```

### 3-2. 첫 번째 URL 색인 요청 (도화살)

**단계 1**: URL 입력 필드 클릭
- 화면 상단의 검색 필드 클릭

**단계 2**: 첫 번째 URL 입력
```
https://muunsaju.com/dictionary/dohwa-sal
```

**단계 3**: Enter 키 또는 검사 버튼 클릭
- 페이지가 로드되고 색인 상태 확인

**단계 4**: 색인 상태 확인
```
✅ 색인이 생성됨 (이미 색인된 경우)
또는
⏳ 색인 요청 (색인되지 않은 경우)
```

**단계 5**: "색인 요청" 버튼 클릭 (필요한 경우)
- "색인 요청" 버튼이 표시되면 클릭
- 또는 "Google에 다시 크롤링 요청" 클릭

**화면 예시**:
```
┌──────────────────────────────────────────────┐
│ URL 검사                                     │
├──────────────────────────────────────────────┤
│                                              │
│ URL 입력: ┌──────────────────────────────┐  │
│          │ https://muunsaju.com/...     │  │
│          └──────────────────────────────┘  │
│                                              │
│ 검사 결과:                                   │
│ ✅ 색인이 생성됨                             │
│                                              │
│ [색인 요청] [세부정보 보기]                  │
└──────────────────────────────────────────────┘
```

### 3-3. 추가 URL 색인 요청 (선택사항)

**빠른 방법**: 위 3-2 단계 반복
```
다음 URL들도 동일하게 요청:
- https://muunsaju.com/dictionary/gab-mok
- https://muunsaju.com/dictionary/eul-mok
- https://muunsaju.com/dictionary/byeong-hwa
- ... (필요한 만큼)
```

**일괄 요청 방법** (선택사항):
1. Sitemap 제출 후 Google이 자동으로 크롤링
2. 모든 URL을 수동으로 요청할 필요는 없음
3. 중요한 URL 5-10개만 수동 요청 권장

## 4. 색인 상태 모니터링

### 4-1. Sitemap 색인 상태 확인

**확인 위치**: 좌측 메뉴 → "Sitemaps"

**확인 항목**:
```
┌─────────────────────────────────────────┐
│ Sitemap: https://muunsaju.com/sitemap.xml
├─────────────────────────────────────────┤
│ 상태: ✅ 성공                             │
│ 발견된 URL: 49개                         │
│ 색인된 URL: 0개 → 5개 → 15개 → ...      │
│ 마지막 다운로드: 2026-02-12 18:45:32    │
└─────────────────────────────────────────┘
```

### 4-2. 색인 진행 상황 추적

**타임라인**:
- **즉시 (0-1시간)**: Sitemap 발견
- **1-3일**: 초기 크롤링 시작
- **3-7일**: 일부 URL 색인화
- **1-2주**: 대부분 URL 색인화
- **2-4주**: 검색 결과 표시 시작

**확인 방법**:
```
1. 매일 같은 시간에 확인
2. "색인된 URL" 개수 변화 추적
3. Google Search Console 이메일 알림 확인
```

### 4-3. 개별 URL 색인 상태 확인

**단계**:
1. URL 검사 도구 접속
2. URL 입력: `https://muunsaju.com/dictionary/dohwa-sal`
3. 검사 결과 확인:
   - ✅ 색인이 생성됨 (이미 색인됨)
   - ⏳ Google에서 크롤링 중
   - ❌ 색인되지 않음

## 5. 자주 묻는 질문

### Q1. Sitemap 제출 후 얼마나 걸리나요?

**A**: 
- **초기 발견**: 1-3일
- **크롤링 시작**: 3-7일
- **색인화**: 1-2주
- **검색 결과 표시**: 2-4주

Google의 크롤링 속도는 사이트 권위성, 업데이트 빈도 등에 따라 다릅니다.

### Q2. 모든 49개 URL을 수동으로 요청해야 하나요?

**A**: 아니요. 다음 방법 중 선택하세요:

**방법 1: Sitemap만 제출** (권장)
- Sitemap 제출 후 Google이 자동으로 크롤링
- 시간이 더 걸리지만 편함

**방법 2: 중요 URL 수동 요청**
- 도화살, 갑목, 을목 등 5-10개 중요 URL 수동 요청
- 나머지는 Sitemap으로 자동 크롤링

**방법 3: 모든 URL 수동 요청**
- 가장 빠르지만 시간 소요
- 필요하면 수행

### Q3. "색인 요청" 버튼이 안 보이면?

**A**: 다음 경우 중 하나입니다:

**경우 1: 이미 색인됨**
```
✅ 색인이 생성됨
→ 이미 색인되어 있으므로 추가 요청 불필요
```

**경우 2: 최근에 요청함**
```
⏳ Google에서 크롤링 중
→ 24시간 후 다시 요청 가능
```

**경우 3: 색인 불가**
```
❌ 색인되지 않음
→ 이유 확인 후 문제 해결 필요
```

### Q4. 색인 요청 후 검색 결과에 나타나려면?

**A**: 다음 단계를 거칩니다:

1. **색인화**: URL이 Google 데이터베이스에 저장됨
2. **순위 결정**: 관련 키워드에 대한 순위 계산
3. **표시**: 검색 결과에 표시 (보통 1-4주)

**검색 결과 확인 방법**:
```
Google 검색: site:muunsaju.com/dictionary/dohwa-sal
또는
Google 검색: "도화살" site:muunsaju.com
```

### Q5. Sitemap이 너무 크면?

**A**: muunsaju.com의 경우 49개 URL이므로 문제없습니다.

**참고**:
- Sitemap 최대 크기: 50MB
- 최대 URL 개수: 50,000개
- muunsaju.com: 49개 (매우 작음) ✅

### Q6. Sitemap 업데이트 시 다시 제출해야 하나요?

**A**: 자동 감지됩니다.

**자동 감지**:
- Google이 주기적으로 Sitemap 확인
- 새로운 URL 자동 발견
- 수동 재제출 불필요

**수동 재제출** (선택사항):
- 급히 색인화하고 싶을 때
- 위 2-3 단계 반복

## 📞 문제 해결

### 문제 1: Sitemap 제출 후 "오류" 표시

**원인**: 
- Sitemap URL이 잘못됨
- Sitemap 파일이 없음
- 접근 권한 문제

**해결**:
1. Sitemap URL 확인: `https://muunsaju.com/sitemap.xml`
2. 브라우저에서 직접 접속 테스트
3. 개발팀에 Sitemap 파일 확인 요청

### 문제 2: URL 색인 요청 후 변화 없음

**원인**:
- Google 크롤링 대기 중
- 페이지에 문제 있음

**해결**:
1. 1-3일 기다린 후 다시 확인
2. "세부정보 보기" 클릭하여 문제 확인
3. 개발팀에 보고

### 문제 3: 2주 후에도 색인되지 않음

**원인**:
- robots.txt에서 차단됨
- Meta robots 태그 문제
- 페이지 품질 문제

**해결**:
1. Google Search Console의 "색인 생성" → "페이지" 확인
2. 문제 메시지 확인
3. 개발팀에 보고
